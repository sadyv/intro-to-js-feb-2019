Git

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

It allows you to maintain multiple versions of a codebase

  • Process to save all modified files.

    • Stage. Stage files that you want to save

      staging is going to allow you to create a commit with the staged files. needed to allow commit

      example

      git add .

    • Commit. Commit files that you want to save

      commit allows you to package staged files together. needed to push changes.

      example

      git commit -m "feat(index) create index.html"

    • Push. Push merges what you have committed to the destination branch

      example

      git push origin master

  • Process to clone a repo.

    • Clone. Makes well... a clone. locally.

      origin is set to the online repo

      example

      git clone URL
      *replace url with repo (https url or ssh url)

  • Process to fork a repo.

    • Fork. Puts a fork in it ( your github account is "it" ).

      places a fork, which is like a clone, on your github account

      example

      *just go to the repo you want to make a version of and click FORK at the top. Easy..
